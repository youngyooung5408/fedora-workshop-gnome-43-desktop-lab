## version 1.2.4

For this version, add a workflow rule that keeps `aesthetic preference.md` synchronized with newer user preferences and durable task features.

- If a newer user preference differs from the existing aesthetic direction, update `aesthetic preference.md` so later work follows the user's latest preference.
- If a new task introduces a durable key desktop, UI, or interaction feature, add it to `aesthetic preference.md` for later workflow use.
