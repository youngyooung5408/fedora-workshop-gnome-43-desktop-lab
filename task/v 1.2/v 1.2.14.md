## Diagnosis

The v1.2.13 code listened for fullscreen changes, but its primary test used an
older display method and otherwise depended on the focused window. GNOME Shell
49 exposes the authoritative fullscreen state on the layout monitor itself as
`monitor.inFullscreen`, which is also what the bundled GNOME Shell window-list
extension uses. The dock must check that state first so suppression continues
even when focus changes.

## Fix

- Check the dock monitor's `inFullscreen` property first.
- Keep the older display query and focused-window test only as compatibility fallbacks.
- Preserve the v1.2.13 bounded reveal zone and removed bottom gesture zone.
