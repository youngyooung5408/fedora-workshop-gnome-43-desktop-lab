## version 1.2.17

# Feature-scoped host installation

The VM is a development and comparison environment, not a byte-for-byte host
template. Store every host-approved feature and its owned surface in one
registry. Each release selects exact feature revisions; everything outside
those registered surfaces remains as the computer already had it.

The previous workflow silently carried v1.2.10's coarse three-extension
manifest into every later release. That omitted the accepted window-button
order while authorizing unchanged Bluetooth Battery and Codex Usage directories
to be recopied.

For v1.2.17:

- Register Bluetooth Battery, Codex Usage, Desktop Lab v1.2, and window-button
  order as separate features.
- Manage only `org.gnome.desktop.wm.preferences button-layout` for the button
  order, with value `close,maximize,minimize:`.
- Compare live extension payload hashes and skip byte-identical features.
- Block unknown local modifications inside a managed extension.
- Permit deliberate application of an older registered release without
  changing features that release does not own.
- Preview and back up every change, restore all affected feature surfaces on
  failure, and preserve all unregistered host state.
- Generate manifests only from an explicit release registry entry; never infer
  host scope from the complete VM profile or silently carry a manifest forward.
