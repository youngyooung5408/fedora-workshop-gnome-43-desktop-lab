## version 1.2.15

  # Left dock behavior with maximized and tiled windows

  The current fullscreen behavior is good and must be preserved.

  The left-side dock currently hides during fullscreen applications, but it
  still appears over maximized windows and tiled windows. I want to improve this
  behavior.

  ## Desired behavior

  Use the active window and the dock's monitor to determine whether the dock
  should be hidden.

  - When the active window is maximized, hide the left dock.
  - When the active window is tiled on the left half of the screen, hide the
  left dock.
  - When the active window is tiled on the right half of the screen, allow the
  left dock to remain available.
  - When the active window is restored to a normal window, restore the normal
  hidden-dock behavior.
  - When a left-tiled window becomes right-tiled, allow the dock to appear
  normally.
  - When a right-tiled window becomes left-tiled, hide the dock.
  - If the active window changes, reevaluate the dock state immediately.
  - If the workspace changes, reevaluate the dock state for the current
  workspace.
  - The dock must not remain visibly stuck after maximize, unmaximize, tile,
  untile, focus, or workspace changes.
  - Preserve the existing fullscreen behavior exactly: fullscreen applications
  must continue to suppress the dock and its edge-reveal zone.

  ## Preferred interpretation

  Treat a maximized window or a left-tiled window as occupying the dock area.

  Treat a right-tiled window as not occupying the dock area, so the left dock
  remains usable.

  The solution should use GNOME Shell's authoritative window state and frame
  geometry where possible. Do not rely only on pointer motion or the currently
  focused window if a more reliable Shell signal is available.

  ## Acceptance checks

  Verify that:

  1. Fullscreen applications still hide the dock.
  2. A maximized application hides the dock.
  3. Restoring a maximized application allows normal dock behavior again.
  4. A left-tiled application hides the dock.
  5. A right-tiled application does not force the dock to hide.
  6. Switching between left-tiled and right-tiled states updates the dock
  immediately.
  7. Switching focus between windows updates the dock correctly.
  8. Changing workspaces does not leave the dock stuck in the wrong state.
  9. An already open or pinned dock is closed when a maximized or left-tiled
  window becomes active.
  10. The existing dock reveal animation, bounded reveal area, app clusters, and
  fullscreen suppression continue working.
